MISSION BRIEF

What you're building

A WINWING PTO2 panel — landing gear lever, flaps, engine switches, and a guarded red JETTISON button — plugs into a Raspberry Pi. One Python program reads every switch, plays a cinematic callout with your kids' names in it, drives the panel's physical warning lights, and serves a full-screen bridge to any TV. After setup it runs entirely offline. Power on the Pi, and the room is a spaceship.

PRE-FLIGHT

The build, in six steps

A relaxed afternoon. The full guide covers every command, flashing the SD card, audio config, and troubleshooting.

Flash the Pi

Raspberry Pi OS (64-bit) via Raspberry Pi Imager. Set hostname, Wi-Fi, and SSH in the imager's settings screen.

Clone the project

git clone https://github.com/michaelditter/open-cockpit.git ~/open-cockpit
cd ~/open-cockpit/cockpit

Name your ship & crew

Edit config.json — ship name, kids' names, callsigns, mottos. This one file drives the bridge display and the voice pack.

Generate the voice pack

36 cinematic lines rendered with your crew's names. Pick any ElevenLabs voice — booming trailer narrators work brilliantly. (No ElevenLabs? Record grandpa instead. Honestly better.)

export ELEVENLABS_API_KEY=sk_...
python3 generate_sounds.py --dry-run   # preview the scripts
python3 generate_sounds.py             # render ../sounds/*.mp3

Install & map your switches

bash install.sh                              # deps + MP3→WAV for low latency
sudo cp 99-winwing.rules /etc/udev/rules.d/  # LEDs without root (then replug)
.venv/bin/python panel_sounds.py --wizard    # press a switch for each sound

PTO2 owners can skip the wizard — the repo ships pre-mapped for it.

Fly, then make it permanent

.venv/bin/python panel_web.py    # bridge at http://cockpit.local:8080
bash install_kiosk.sh && sudo reboot   # boots straight into the cockpit forever

COMMS

FAQ

Do I need a gaming PC or a flight simulator?

No. That's the point. The panel plugs into the Pi and the Pi does everything — sounds, lights, display. (USB is point-to-point, so if you also want the panel to fly a sim on a PC, run the sounds script on that PC instead.)

Does it work with panels other than the WINWING PTO2?

For sounds and the bridge: yes — any USB HID device the Linux kernel exposes (button boxes, throttles, joysticks). Run panel_sounds.py --wizard to map it. Physical LED control is PTO2-specific today; ports to other panels are welcome pull requests.

What does it cost to run?

Nothing recurring. The ElevenLabs voice generation is a one-time job (a starter plan covers the whole 36-line pack; regenerate whenever you like). After that the cockpit is fully offline — optional space imagery loads from Wikimedia when online.

How hard is the build, really?

If you can flash an SD card and paste terminal commands, you can build this. There's no soldering, no wiring, no 3D printing. The hardest part is not spoiling the secret combos.

Can I write my own voice lines?

Yes — voice_lines.json holds all 36 scripts with {c1}/{c2}/{c3}/{ship} name tokens. Edit any line and re-render just that one with generate_sounds.py --only gear_up. Or skip AI entirely and record your own MP3s.

What's next for the project?

A voice-interactive ship computer (push-to-talk → speech-to-text → LLM → ElevenLabs, aware of ship state), a stateful mission engine with per-kid ranks and weekly AI-generated missions, live ISS and launch data on the bridge, and WLED room-lighting sync. Star the repo to follow along.
